# Contribution Notes

This section is the knowledge base for contributors to the HANA Crossplane provider. It collects architectural decisions, implementation patterns, and essential background knowledge that goes beyond what the code itself communicates.

Whether you are onboarding as a new contributor or revisiting a non-obvious design choice — this is the place to look first.

## What you'll find here

- **Architecture Decision Records (ADRs):** Documented reasoning behind significant design and implementation decisions, so the "why" is never lost.
- **Best practices:** Patterns and conventions established by the team for consistency and maintainability across the provider.
- **Developer guides:** Focused how-tos for specific technical areas covering provider-internal mechanisms.

## Guides

- [X.509 Certificate-Based Authentication](/docs/crossplane-provider-hana/docs/contribution-notes/20250822-hana-x509-authentication) — ADR covering the decision to adopt X.509 certificate-based user authentication in the HANA provider.
- [Audit Policy CRD](/docs/crossplane-provider-hana/docs/contribution-notes/20250826-adr-audit-policy-crd) — ADR for the audit policy custom resource definition.
- [Instance Mapping Architecture](/docs/crossplane-provider-hana/docs/contribution-notes/20260218-instance-mapping-architecture) — Architecture decision covering how HANA Cloud instances are mapped and referenced within the provider.
